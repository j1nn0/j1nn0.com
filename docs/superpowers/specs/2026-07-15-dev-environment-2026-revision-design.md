# 開発環境記事の改稿設計

## 目的

`content/posts/dev-environment-2026.md` を、2026年時点の開発環境を紹介する体験記事として正確かつ読みやすくする。

## 変更範囲

- タイトルを「2026年の開発環境」に変更する。
- 見出しを「VS Code をやめた理由」に変更し、本文の理由の数と矛盾させない。
- `devcontainer CLI` を選ぶ理由を、AIエージェントと相性がよい点に限定して記述する。
- Ghostty の性能評価は、客観的な機能と自分の体感を区別する。
- 価格、発送予定、OS、セキュリティ事案など変動しうる情報に、確認時点または具体的な対象を添える。
- `mise` が扱う対象と、集約を迷う理由を明確にする。

## 非変更範囲

- デスク周りの節と画像は維持する。
- 新しい画像、構成図、外部リンク一覧は追加しない。
- 記事の結論である「設定の少ないツールを選ぶ」は変えない。

## 検証

- Hugo ビルドを実行する。
- 記事の書式と禁止表現を検索する。
- 変更した外部仕様の記述を公式ドキュメントと照合する。
